Codex Telegram Bridge

텔레그램 봇, 작업 폴더, Codex CLI 연결을 한 화면에서 설정하는 로컬 브리지입니다.

1. 봇 토큰 입력
2. 허용 사용자 지정
3. 작업 폴더 선택
4. 진단 확인 후 바로 사용

브리지 설정

이제 템플릿은 말투만 바꾸는 프리셋이 아니라 출력 구조까지 바꾸는 기능으로 동작합니다.

빠른 안내 `Allowed User IDs`에는 본인 텔레그램 숫자 ID를 넣고, `Workspace Path`에는 Codex가 작업할 프로젝트 폴더를 지정하세요.
텔레그램에서 구분하기 쉬운 이름으로 정하면 됩니다.
저장된 토큰이 있으면 가려진 상태로 표시됩니다. 비워두고 저장하면 기존 토큰을 유지합니다.
쉼표로 여러 명을 넣을 수 있지만, 1차는 개인 사용 기준을 권장합니다.
드라이브 전체가 아니라 실제 프로젝트 루트 폴더를 지정하는 편이 좋습니다.
자동 탐지가 되지 않을 때만 `codex.cmd` 또는 `codex.exe` 경로를 넣으세요.
템플릿마다 강제되는 응답 섹션 구성이 다릅니다.
기본값은 600초입니다.
화면에서 입력 중인 값만 잠깐 확인할 때 사용하세요.

기본값은 꺼짐입니다. 일반 사용자는 끈 상태로 시작하고, 특정 환경에서만 필요할 때 켜는 편이 안전합니다.

연결 진단

설치 상태와 로그인 여부, 작업 폴더 위험 신호, 현재 봇 서비스 상태를 여기서 확인할 수 있습니다.
